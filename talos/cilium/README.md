# Cilium (CNI bootstrap)

[Cilium](https://cilium.io) replaces both Flannel and `kube-proxy` for this
cluster. Because Talos boots without any CNI, Cilium has to be present *before*
the API server is reachable for normal Helm installs — so the chart is rendered
into the Talos machine config as an inline manifest at bootstrap time.

### Files

- `values.yaml` — Helm values (kube-proxy replacement on, Hubble + UI on,
  Gateway API on, and the Talos-specific cgroup/security knobs).
- `patch.yaml` — header that disables Talos's built-in kube-proxy and opens an
  `inlineManifests[].contents` block. The renderer indents the Helm output
  underneath it.
- `manifest.gen.yaml` — generated, gitignored, regenerated by
  `just render-cilium` (or implicitly by `just genconfig`).

### Runtime add-ons

The Hubble UI route, OIDC SecurityPolicy, ServiceMonitor, and Grafana dashboard
live in [`k8s/infra/configs/hubble/`](../../k8s/infra/configs/hubble/) and are
applied by Flux after the cluster is up.
